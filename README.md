# Запуск backend

## 1) Терминал

Для начала нам нужно открыть папку backend в любом редакторе кода и открыть терминал для работы с проектом

## 2) Создаем виртуальное окружение

Виртуальное окружение — это изолированная "коробка" для твоего проекта, где живут только нужные ему библиотеки и их конкретные версии. Если ставить всё глобально, они будут конфликтовать. С виртуальным окружением у каждого проекта своя "коробка" — они не мешают друг другу.

```shell
python -m venv .venv  # установка окружения

.\.venv\Scripts\activate  # активация окружения
```

После установки появится папка `.venv`. А после активации должен появиться маркер, что мы успешно активировали виртуальное окружение.

## 3) Устанавливаем  библиотеки

Все необходимые нам библиотеки и их версии храняться в файле `requirements.txt`.

```shell
pip install -r requirements.txt
```

### 4) Создание и подключение базы данных

Заходим в приложение `pgAdmin` и создаем новую базу данных.

В папке backend есть файл `example.env`. Копируем его содержание и создаем новый файл `.env`, в который и переносим данные. 

Тут будут хранится настройки подключения к базе данных PostgreSQL. Без него сервер не сможет подключиться к БД. Данные для подключения - это секретные данные, которые не должен видеть случайный пользователь. Поэтому они храняться в файле `.env`, который находиться в `.gitignore`.

| Переменная | Описание |
|---|---|
| `POSTGRES_DB` | Название базы данных |
| `POSTGRES_USER` | Имя пользователя PostgreSQL (обычно `postgres`) |
| `POSTGRES_PASSWORD` | Пароль пользователя |
| `POSTGRES_HOST` | Адрес сервера БД (обычно `localhost`) |
| `POSTGRES_PORT` | Порт PostgreSQL (по умолчанию `5432`) |

Самому нужно заполнить название базы данных (имя только что созданный базы) и пароль пользователя для подлкючения.

### 5) Создаем миграции для базы данных (если их нет) и применяем

Так как в проекте мы используем ORM для работы с базой данный, то нам необходимо синхронизировать Python-модели с реальной структурой таблиц - то есть сделать миграции

```shell
python manage.py makemigrations  # создаст миграций, если их еще нет
python manage.py migrate  # применит миграции к бд
```


### 6) Создаем суперпользователя (для админки сайта)

Для того, чтобы заполнять данные на сайте, нам неообхоимо зарегистрировать админа.

```shell
python manage.py createsuperuser
```

После необходимо заполнить данные. Советую:
- Логин - admin
- Пароль - 1234
- Почта - можно пропустить, нажав `enter`

### 7) Запускаем сервер

```shell
python manage.py runserver
```

Теперь перейдя по ссылке http://127.0.0.1:8000/ будет располагаться ваш backend

Чтобы открыть админку, необходимо перейти по ссылке http://127.0.0.1:8000/admin/ и указать данные которые вы вводили при создании суперпользователя

Если перейти по ссылке http://127.0.0.1:8000/api/swagger/ откроется окно со всеми ручками вашего backend

# Запуск frontend

### 1) Терминал

Точно также как с backend, нам необходимо открыть редактор кода в папке frontend и запустить терминал. Советую открыть второе окно для наглядности.

### 2) Устанавливаем библиотеки

Если node.js был успешно установлен, то с установкой зависимостей не должно быть проблем. 

Тут нам отдельно создавать виртуальное окружение не нужно.

```shell
npm install
```
Должна появиться папка node_modules

### 3) Создаем файл `.env`

Также как и в backend создаем файл `.env` и копируем в него данные из `example.env`. 

| Переменная | Описание |
|---|---|
| `REACT_APP_PATH_URL_API` | URL API backend'а (например `http://127.0.0.1:8000/api`) |
| `REACT_APP_BASE_URL` | Базовый URL сервера, используется для загрузки изображений (например `http://127.0.0.1:8000`) |

Тут ничего менять не нужно, оставляем стандартные значения

### 4) Запускаем приложение

```shell
npm start
```
Теперь по ссылке http://127.0.0.1:3000/ будет располагаться ваш frontend сайта
